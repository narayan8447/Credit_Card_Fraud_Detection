# Credit Card Fraud Detection

A Streamlit portfolio app that predicts whether a credit card transaction is likely to be fraudulent. The project uses a hybrid machine learning pipeline: an autoencoder compresses the original transaction features, and a Random Forest classifier combines those encoded features with contextual transaction inputs.

## Demo

Run the app locally:

```bash
streamlit run app.py
```

## Project Structure

```text
.
├── app.py                  # Streamlit user interface
├── utils.py                # Model loading and inference helpers
├── train_and_save.py       # Training script for the hybrid model
├── models/                 # Saved model artifacts
│   ├── feature_means.csv
│   ├── encoder_model.h5
│   ├── hybrid_scaler.joblib
│   └── rf_hybrid_model.joblib
├── notebooks/              # Exploratory notebook
├── requirements.txt        # Deployment dependencies
└── reports/                # Training reports generated by train_and_save.py
```

## Dataset

The training script expects `creditcard.csv` in the project root. The dataset is the public credit card fraud dataset commonly used from Kaggle. Because the CSV is large, it is ignored by Git and should be downloaded separately before retraining.

## Setup

Python 3.11 is recommended for TensorFlow compatibility.

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## Training

```bash
python train_and_save.py
```

Training saves:

- `models/encoder_model.h5`
- `models/hybrid_scaler.joblib`
- `models/rf_hybrid_model.joblib`
- `models/feature_means.csv`
- `reports/training_report.txt`

## GitHub Deployment Notes

`models/rf_hybrid_model.joblib` is larger than GitHub's normal 100 MB file limit. Use Git LFS for model artifacts before pushing:

```bash
git lfs install
git lfs track "models/*.joblib" "models/*.h5"
git add .gitattributes models/
```

The raw dataset `creditcard.csv` is intentionally ignored and should not be committed.

## Notes

- The app is designed for demonstration and portfolio use.
- The contextual features in the current training script are synthetically generated, because they are not part of the original public dataset.
- For production fraud detection, the model should be retrained with real behavioral, merchant, device, and location features.
